Сервис для курсовой работы

Описание проблемы. 

Существуют разные фреймворки управления корпоративной архитектурой. 
Одним из них является архитектурный фреймворк TOGAF, развиваемый консорциумом The Open Group.  
Он представляет собой методологию для организации архитектурной практики на предприятии, 
и может быть адаптирован к использованию в самых различных компаниях.

Архитектурный фрейймворк TOGAF базируется на комплексном подходе к корпоративной архитектуре, включающей три основные домена: 
- бизнес-архитектура
- архитектура информационных систем (архитектура приложений и архитектура данных)
- технологическая архитектура

Его основным недостатком можно считать необходимость "приземления" общих абстрактных понятий и методологии 
в практическую плоскость, позволяющую использовать его в повседневной работе по развитию корпоративной архитектуры. 

Предлагаемое решение.  

Нужен сервис, который позволит: 
1) описывать архитектуру предприятия с помощью реляционной модели данных
2) визуализировать отношения между архитектурными объектами с разным уровнем абстракции на основе этой модели

В качестве полезных дополнительных функций данного сервиса можно рассмотреть следующие:
- архитектурный глоссарий
- доступ к материалам архитектурного репозитория в базе знаний (wiki) 
- возможность интеграции с трекерами задач (системы управления проектами и сервис-дески) 

Содержание курсовой работы по курсу Базы данных 

Разработка упрощенной модели данных описания корпоративной архитектуры с учетом:
1) методологии фреймворка TOGAF,
2) необходимости конкретизации архитектурных объектов и связей между ними. 

Необходимо заложить в структуру архитектурных данных следующие объекты: 
- глоссарий
- объекты бизнес-архитектуры: 
	-- функциональная структура предприятия: 
		-- бизнес-функции
		-- бизнес-процессы
	-- оргструктура
		-- компании
		-- подразделения 
		-- персонал
- объекты и связи архитектуры информационных систем: 
	-- архитектура приложений: 
		-- приложения
		-- компоненты приложений
		-- функции приложений		
	-- архитектура данных и ее связи с архитектурой приложений 
		-- справочные и мастер-данные
		-- транзакционные данные 
	-- архитектура интеграции 
		-- потоки данных 
		-- сервисы
- объекты технологической архитектуры: 
	-- оборудование
		-- серверы
		-- сети и телекоммуникационное оборудование
		-- персональные компьютеры
		-- периферийное оборудование
	-- программное обеспечение: 
		-- системное ПО
		-- технологическое ПО
		-- ПО общего назначения
	-- технологические компоненты
- размещение данных, приложений и их компонент в объектах технологической архитектуры (опционально) 
 
Многовато для курсовой, конечно, но попробую уложиться.
